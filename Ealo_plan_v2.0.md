# 모님(Mornim) 서비스 기획서 v2.0

**Morning + Affirmation — 매일 아침 성공의 말로 하루를 여는 습관**

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 모님 (Mornim) |
| 네이밍 어원 | Morning + Affirmation |
| 핵심 슬로건 | "말하면, 이루어진다." |
| 서비스 형태 | PWA(웹앱) MVP → 모바일 앱 전환 |
| 서비스 목적 | 매일 아침 성공의 말로 하루를 열어 작은 성공에서 큰 성공을 이룰 수 있도록 도와주는 앱 |

---

## 2. 서비스 철학

- 앱을 열 때마다 "나는 할 수 있다"는 에너지를 느껴야 한다
- 모든 인터랙션은 반드시 성취감으로 마무리된다
- 비교 없이, 경쟁 없이, 오직 나를 응원하는 공간
- 부정적인 말은 AI가 부드럽게 긍정의 말로 바꿔준다
- 무겁지 않게, 게임처럼 가볍게, 그러나 진심 어리게

---

## 3. 타겟 사용자

- 꿈을 이루려는 사람
- 매일 작은 성취감을 원하는 사람
- 부정적 자기대화를 반복하는 20–40대
- 자기계발을 시작하고 싶지만 쉽게 도전하지 못하는 사람
- 명상·마음챙김 앱을 써봤지만 지속이 어려웠던 사람

## 4. 핵심 가치 (모님 8가지)

| 가치 | 설명 |
|------|------|
| 간결함 | 앱을 열면 바로 확언을 말할 수 있어야 한다 |
| 성취감 | 즉각적인 칭찬, 시각·청각 보상이 따른다 |
| 지속성 | 달력으로 나의 성장 흔적이 쌓이는 것을 눈으로 확인한다 |
| 누구나 | 성공의 말이 생각나지 않아도 채팅으로 확언을 추천해준다 |
| 함께 | 친구들과 함께 서로를 격려한다 (P3) |
| 가벼움 | 게임처럼 즐기다 보면 확언이 자연스럽게 내 삶에 스민다 |
| 보호 | 부정적 말이 나와도 AI가 부드럽게 긍정으로 전환해준다 |
| 프라이버시 | 로그인 없이 기기에 데이터를 저장하며 시작할 수 있다 |

## 5. 플랫폼 전략

```
플랫폼: PWA(웹앱) → React Native 앱
결정 이유: MVP를 빠르게 배포해 핵심 기능과 사용자 반응을 먼저 검증한다
```

**화면 크기 기준**

- 기준 뷰포트: 390 × 844px (iPhone 14 기준)
- 최대 컨테이너: max-width 430px, margin 0 auto
- 데스크톱: 중앙 정렬 + 양쪽 여백 처리 (앱처럼 보이도록)

## 6. 로그인 전략

로그인 없이 앱의 핵심 경험 전체를 사용할 수 있다. 로그인은 P3에서 커뮤니티 기능과 함께 도입한다.

```
P0 ~ P2 (로그인 없음)
  → 모든 데이터 기기 localStorage에 저장
  → OpenAI API 호출은 익명으로 처리
  → 핵심 기능 전체 사용 가능

P3 (로그인 도입)
  → 구글 원탭 로그인
  → Supabase로 기존 로컬 데이터 마이그레이션 (클라우드 동기화)
  → 커뮤니티 기능 활성화
  → 멀티 디바이스 지원
```

## 7. 온보딩 경험

**목표: 로그인 없이, 설명 없이, 바로 핵심 경험부터**

```
앱 첫 실행
  → 화면 전체에 한 문장: "오늘 나에게 해주고 싶은 말이 있나요?"
    → 예시 확언 3개가 말풍선으로 떠오름 (탭하면 선택)
      → 선택한 확언이 동적 텍스트로 화면 가득 표시
        → 스와이프
          → 전체 화면 카메라 + 확언 텍스트 오버레이
            → 소리 내어 읽기 → STT 글자 반응 → 완료
              → 칭찬 + "모님에 오신 걸 환영해요"
                → 바로 앱 사용 시작 (로그인 없음)
```

첫 사용 직후 두 가지만 질문한다.

- "지금 가장 힘든 게 뭔가요?" → 카테고리 8개 중 선택
- "매일 몇 시에 모님을 열고 싶으세요?" → 알림 시간 설정

이 두 답변으로 AI가 첫 확언 3개를 생성해 바로 채워준다.

## 8. 핵심 기능 목록

### P0 — MVP 핵심 (2주)

**기능 1. 오늘의 확언 선택 구조**

저장된 확언 중 오늘 말할 확언 3개를 다음 방식으로 결정한다.

- 기본: 전날 밤 "내일의 나에게" 화면에서 사용자가 직접 내일의 확언 3개를 고른다
- 대체: 전날 선택을 안 했을 경우 AI가 최근 말한 확언 패턴과 카테고리를 고려해 자동으로 3개를 추천한다

**기능 2. 앱을 열고 바로 확언 말하기**

```
앱 실행
  → 어제의 내가 남긴 한 마디 표시 (내일의 나에게 ON 상태일 경우)
    → 오늘의 확언 1개 동적 텍스트로 화면 가득 표시
      → 스와이프
        → 전체 화면 카메라 + 확언 텍스트 오버레이 (완전 투명)
          → 소리 내어 읽기
            → STT 단어 인식 → 글자 하이라이트 반응
              → 문장 전체 인식 완료 시 자동으로 다음으로 넘어감
              → 또는 완료 버튼 탭으로 다음으로 넘어감
                → 칭찬 화면 → 다음 확언으로
```

카메라 권한이 없을 경우 카메라 없이 텍스트만 표시하는 대안 모드를 제공한다.

**기능 3. 하루 확언 구조**

- 기본 미션: 오늘의 확언 3개를 아침에 연달아 말하기
- 추가 선택: 3개 완료 후 "오늘 더 말하고 싶으신가요?" → 하루 중 언제든 1개씩 추가 (최대 4개 추가, 하루 총 최대 7개)
- 저장 가능한 확언 총 개수: 제한 없음 (나의 성장 라이브러리)

**기능 4. 확언 만들기**

세 가지 방법으로 확언을 만들 수 있다.

- 직접 입력: 확언 입력 완료 후 저장 버튼 탭 시 AI 분석 → 부정적 내용 감지 시 "이렇게 바꿔볼까요?" + 대안 확언 카드 표시 → 대안 선택 또는 원본 그대로 저장 → 카테고리 선택
- AI 추천: 고민·감정 입력 → 확언 5개 추천 카드 → 선택 저장
- Talk Mode: 대화 후 → 확언 5개 추천 카드 → 선택 저장

**기능 5. 달력 체크**

하루 완료한 확언 개수를 달력에 진하기로 표현한다.

- 1–2개: 연한 색 (시작)
- 3–5개: 중간 색 (기본 미션 완료)
- 6–7개: 가장 진한 색 (풀 달성)

색상은 그날 가장 많이 완료한 카테고리 색상으로 표시한다. 확언 완료 시 달력이 바로 반영된다.

### P1 — 런칭 후 1차 업데이트 (4주)

**기능 6. 습관 훅 — 스트릭 + 보호막**

매일 확언 3개(기본 미션) 완료 시 스트릭이 쌓인다. 7일 연속 달성 시 보호막 1개를 획득하며, 보호막이 있으면 하루를 쉬어도 스트릭이 유지된다. "오늘 못 했다 → 다 망했다 → 앱 삭제"의 악순환을 방지하는 장치다.

**기능 7. 내일의 나에게 보내는 확언**

설정에서 ON/OFF 선택 가능한 선택 기능이다.

- ON: 오늘 확언 완료 후 화면 표시
- OFF: 확언 완료 후 바로 달력으로

ON 상태일 때 흐름:

```
오늘 확언 완료
  → 내일의 나에게 한 줄 메시지 작성 (선택, 건너뛰기 가능)
  → 저장된 확언 목록에서 내일 말할 확언 3개 선택 (선택, 건너뛰기 가능)
    → 건너뛰면 AI가 자동 추천
```

다음 날 앱을 열면 어제의 내가 남긴 메시지가 먼저 보이고, 내가 직접 고른 확언이 동적 텍스트로 펼쳐진다. 푸시 알림은 "어제의 내가 오늘 너한테 할 말이 있대"로 표시된다.

**기능 8. 주간 성장 리포트**

매주 일요일 밤, 이번 주 내가 가장 많이 말한 확언 한 문장과 달력 히트맵을 보여준다.
예: "이번 주 당신이 자신에게 가장 많이 한 말: 나는 충분히 잘하고 있다."

**기능 9. 게임 (30초–2분)**

- 게임 1 벽돌 깨기: 숨겨진 확언 단어들을 벽돌로 배치 → 공을 튕겨 단어 벽돌을 깸 → 모든 단어 클리어 시 완성된 확언이 화면을 채움 → TTS 낭독 + 칭찬
- 게임 2 단어 정렬: 뒤섞인 단어 카드를 드래그 앤 드롭으로 정렬 → 완성 시 TTS 낭독 + 칭찬
- 게임 3: 추후 기획

게임 완료는 확언을 한 것으로 인정한다.

### P2 — 2차 업데이트 (2주)

- 테마 선택 (웜 미니멀 / 다크 에너지 / 소프트 그린)
- 알림 설정 및 시간 변경
- 통계 (총 확언 수, 연속 달성일, 카테고리별 분포)
- 백업 (로컬 데이터 내보내기)
- 데이터 초기화
- 사용설명서

### P3 — 3차 업데이트 (별도 산정)

- 구글 로그인 (원탭)
- Supabase 클라우드 동기화 (기존 로컬 데이터 마이그레이션)
- 커뮤니티: 친구 목록 + 오늘의 격려
- 익명 확언 피드
- 멀티 디바이스 지원

## 9. 확언 카테고리

| 카테고리 | 예시 확언 |
|------|------|
| 나 자신 | 나는 오늘도 충분히 잘하고 있다 |
| 일과 커리어 | 나는 내가 하는 일에서 가치를 만든다 |
| 돈과 풍요 | 나는 풍요를 받아들일 준비가 되어 있다 |
| 관계와 사랑 | 나는 좋은 사람들을 내 삶으로 끌어당긴다 |
| 건강과 몸 | 나의 몸은 매일 더 강해지고 있다 |
| 용기와 도전 | 나는 두려워도 한 발 내딛을 수 있다 |
| 마음과 평온 | 나는 흔들려도 다시 중심을 찾는다 |
| 오늘 하루 | 오늘은 좋은 일이 나에게 일어난다 |

카테고리는 세 가지 방식으로 활용된다.

- 확언 저장 시 태그로 분류되어 달력 색상에 반영된다
- AI 추천의 기준점이 되어 고민 입력 시 카테고리를 자동 판단해 추천한다
- 온보딩 첫 질문의 선택지로 사용되어 첫 확언을 즉시 제공한다

## 11. 핵심 사용자 흐름

**매일 아침 플로우**

```
앱 실행
  → 어제의 내가 남긴 한 마디 표시 (ON 상태일 경우)
    → 오늘의 확언 1번 동적 텍스트로 화면 가득 표시
      → 스와이프
        → 전체 화면 카메라 + 확언 텍스트 오버레이
          → 소리 내어 읽기 → STT 글자 반응
            → 자동 완료 또는 완료 버튼 탭
              → 칭찬 → 2번 확언으로
                → 3개 완료 → 칭찬 + 달력 체크 ✓
                  → "오늘 더 말하고 싶으신가요?" (추가 선택)
                    → 내일의 나에게 화면 (ON 상태일 경우)
```

**확언이 없을 때**

```
확언 만들기 탭 진입
  → 직접 입력 / AI 추천 / Talk Mode 중 선택
    → 카테고리 선택 → 저장
      → (직접 입력의 경우) AI 부정 감지 → 대안 추천 or 원본 저장
        → 홈에서 말하기
```

## 12. UX/UI 설계 원칙

**동적 텍스트 설계**

확언 문장을 의미 단위로 끊어 줄별로 배치한다. 각 줄은 순서대로 페이드인 또는 슬라이드업으로 등장한다. 핵심 단어는 크고 굵게, 조사와 부사는 작게 표현해 리듬감을 만든다. 배경은 해당 확언의 카테고리 색상으로 부드럽게 표현한다.

예시:
```
나는         ← 18px, 보조색
오늘도       ← 36px, 메인색 (가장 크게)
충분히       ← 20px, 연한색
잘하고 있다  ← 22px, 포인트색
```

**카메라 말하기 화면**

- 전체 화면 카메라 (전면, 거울 효과)
- 확언 텍스트는 하단에 완전 투명 오버레이로 표시
- 텍스트에 text-shadow 처리로 어떤 배경에서도 가독성 확보
- STT 인식된 단어마다 하이라이트 처리
- 완료 버튼 + STT 자동 완료 병행

**색상 역할 정의**

| 용도 | 적용 화면 |
|------|------|
| 테마 메인 포인트 | 완료 버튼, 스트릭, 달력 강조 |
| 테마 서브 포인트 | STT 하이라이트, 진행 표시 |
| 카테고리 8색 | 달력 색상, 확언 태그, 동적 텍스트 배경 |
| 살몬 계열 | 부정확언 감지 대안 추천 UI |
| 차갑지 않은 배경 | 테마별 전체 앱 배경 |

**UX 핵심 원칙**

- 시각·청각 보상을 철저히 구현한다
- 모든 선택 기능은 건너뛸 수 있어야 한다
- 채팅은 따뜻하면서 도전적인 느낌으로 한다
- 게임은 진지하지 않게, 30초–2분으로 끝낸다
- 달력은 나의 성장 일지다
- 부정적 표현은 "틀렸어요"가 아니라 "이렇게 바꿔볼까요?"로 전환한다
- 마이크로인터랙션을 적극적으로 설계한다

## 13. 기술 스펙

**MVP — PWA**

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js + Tailwind CSS |
| AI 기능 | OpenAI API (확언 추천, 부정 감지, Talk Mode) |
| 음성 | Web Speech API (STT 글자 반응 + TTS 칭찬) |
| 데이터 저장 | localStorage (P0~P2, 로그인 불필요) |
| 배포 | Vercel |

**P3 전환 시 추가**

| 영역 | 기술 |
|------|------|
| 인증 | Supabase Auth (구글 원탭) |
| 데이터베이스 | Supabase (로컬 데이터 마이그레이션) |
| 앱 전환 | React Native (Expo) |
| 상태관리 | Zustand |

**비용 구조**

| 항목 | 비용 |
|------|------|
| Next.js / Tailwind CSS | 무료 (오픈소스) |
| Vercel | 무료 플랜으로 MVP 운영 가능 |
| localStorage | 무료 (브라우저 내장) |
| Web Speech API | 무료 (브라우저 내장) |
| OpenAI API | 사용량 기반 과금 |
| Supabase | P3 도입 시, 무료 플랜으로 시작 가능 |

오프라인 환경을 대비해 PWA Service Worker로 기본 확언 캐싱을 구현한다.
